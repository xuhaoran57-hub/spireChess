# R1-S0-06 B03 Token 繁茂改造分析

## 结论

本轮改造有效修复了频率召唤的 Token 成长链。B03-N 从几乎无胜算提升到可观察但仍偏弱；B03-H 已进入合理竞争区间，但对 B04-H 形成了过强克制，需要在后续 S1 中重点观察。

本报告来自无 Unity 环境下的 .NET 8 预演，不替代正式 Unity 基线。

## 候选改动

- 狐群巢母恢复为普通 4/5、金色 8/10。
- 群枝唤灵者普通与金色召唤强化允许作用于 Token。
- 万蹄奔潮普通与金色召唤攻击强化允许作用于 Token。
- 金色万蹄的溅射允许作用于 Token。
- 藤冠祭司新增繁茂：两个友方随从死亡后永久获得 1 层，每场最多一次；普通最多 4 层，金色最多 8 层；召唤 Token 时赋予等同层数的本场攻击。
- B03 的 S4 由裂爪幼兽替换为藤冠祭司，固定繁茂为 N=2、H=4。

## 机制表现

- 机制压力场景的一轮原始输出：N 从 82.43 提升到 108.02，增加 31.0%；H 从 164.86 提升到 252.83，增加 53.4%。
- B03 的平均立即攻击次数基本保持在每场约 3.7 至 4.6 次，说明强度提升主要来自 Token 攻击质量，而不是额外制造攻击次数。
- 所有 B03 汇总对局的召唤失败均为 0。
- 藤冠在竞争对局中平均每场获得约 0.80 至 0.98 层繁茂，符合“两个友军死亡、每场最多一次”的预期。
- 机制级断言确认：1/1 Token 在普通群枝、普通万蹄和 3 层繁茂作用后成为 7/2，并且同一召唤事件只产生一次立即攻击。

## 判断与下一步

B03-H 的整体得分率 45.65%，暂不建议继续做全档位增伤。B03-N 仍以 12.35% 排名末位，但应先进入 S1 扩大样本，避免基于 100 种子继续叠加数值。

后续重点观察：

1. B03-H 对 B04-H 的 99.5% 是否在 S1 仍然稳定；若稳定，应优先削减该对局的克制极化，而不是削弱整套频率召唤。
2. B03-N 对 B01/B02/B05 仍明显偏弱；若 S1 复现，可只调整 N 档夹具的初始繁茂或中期身材，不改 H 档。
3. 在 Unity 中补跑 EditMode 与正式 S0，确认资源加载、序列化和战斗回放表现与 .NET 预演一致。
