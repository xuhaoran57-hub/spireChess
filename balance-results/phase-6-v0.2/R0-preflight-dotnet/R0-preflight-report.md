## 机制压力板观察

- 十二个机制压力板场景最终均为玩家侧零存活。
- `B03_SUMMON` 的非 Token 核心存活率低于 40%，同时其镜像整体得分率远低于 40%，未通过召唤构筑专项条件。
- 本轮没有发现效果无限循环、异常或确定性漂移。
- 压力板主要用于触发并观察机制，不应直接按胜负评价构筑；但全灭结果表明压力强度和核心生存都需要在正式调数前复核。

## R1 建议调查顺序

1. 先确认固定构筑快照和 D01 压力板是否代表预期战斗条件。大量接近 0%/100% 的结果过于系统化，不适合直接用小幅数值调整处理。
2. 优先检查 B03 的召唤顺序、战斗格占用、立即攻击收益和非 Token 核心生存。
3. 拆分 B01/B02 的共同护盾引擎、失盾收益和基础面板影响，再选择单一调数轴。
4. 复核 B05 的标准法术投入与下一战覆盖层，其正常发育首轮输出已经越过专项门槛。
5. 调整 D00 的统计口径：只测首轮或从竞技回合上限中排除，避免制造虚假安全告警。

## 结果适用边界

本报告来自无界面的 `.NET 8.0.20` 预演，适合发现夹具错误、确定性问题、安全问题和大幅平衡离群，但它不是 Unity 正式 R0：

- 没有运行 Unity EditMode/PlayMode 测试；
- 没有验证 Unity/Mono 随机序列；
- 没有完成 20 局三层人工单局；
- 不能与后续 Unity R0 数据直接合并。

正式调数前仍需在 Unity 2022.3.62f3c1 下重跑同一套种子和夹具。
