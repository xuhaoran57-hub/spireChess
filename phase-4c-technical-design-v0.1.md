# 阶段 4C 三层完整单局技术方案 v0.1

版本：0.1
状态：已实现并完成自动化验证；Unity 人工验收待集中执行
上位设计：`phase-4-node-design-v0.1.md`
前置实现：`phase-4a-technical-design-v0.1.md`、`phase-4b-technical-design-v0.1.md`

## 1. 目标与范围

4C 将 4B 单层原型扩展为可通关或失败的三层完整单局：

`第一层 → Boss 奖励 → 第二层 → Boss 奖励 → 第三层 → 最终结算`

本阶段包含三层固定地图、逐层遭遇强度、第一/二层 Boss 三选一、跨层状态保留、最终统计和最小 UI。随机地图、遗物、Boss 专属主动技能、正式平衡与人工时长验收不在本次代码范围内。

## 2. 地图与内容

- 第一层保留 4B 的 7 节点强制首战地图。
- 第二、三层各使用 6 节点地图：普通/精英双起点，分别连接事件或恢复/锻造，最后汇入固定 Boss。
- 每层实际访问 3-4 个节点；每条路线均能到达唯一 Boss。
- 第二层突出溅射和站位压力，第三层突出亡语与残局铺场。
- 事件、锻造和恢复配置跨层复用；战斗遭遇与普通奖励按层独立配置。

## 3. 跨层状态机

新增 `RunPhase.FloorComplete`。第一、二层 Boss 胜利流程：

1. 战斗结算并记录 Boss 胜利。
2. 生成 `RewardCompletionMode.FloorComplete` 的 Boss 三选一。
3. 选择或跳过后完成 Boss 节点并进入 `FloorComplete`。
4. `ContinueToNextFloor()` 从 `IMapProvider` 获取下一层地图，重建 `MapProgressState` 并进入 `MapSelection`。

跨层保留生命、最大生命、`RunTurn`、商店经济时钟、酒馆等级、阵容、永久强化、有限牌池、延迟资源和卡牌奖励队列。旧地图进度不再可操作。第三层 Boss 胜利直接进入 `RunWon`。

## 4. Boss 奖励

第一、二层 Boss 使用不同奖励表，均从不同类别中物化 3 项并允许跳过：

- 不高于当前酒馆等级 +1 的随从；
- 本层对应高价值法术；
- 下次升级费用 -3；
- 下个商店 +5 金币；
- 指定战斗区随从永久 +2/+2。

随从候选继续使用有限牌池预留；选中后进入 FIFO 暂存区，未选或跳过时返还。第三层 Boss 不生成奖励。

## 5. 单局统计与结算

`RunStatistics` 记录战斗胜利/未胜利、精英击败、Boss 击败、三连次数和起止时间。统计只在节点尝试首次结算时提交，重载或重复回调不能重复累计。`RunWon`、`RunLost` 展示到达层数、战绩和最终阵容摘要。

## 6. UI

`RunTest` 继续作为三层地图与选择场景：

- 显示当前层数、生命和累计战绩；
- `FloorComplete` 显示进入下一层按钮；
- 切层后重建节点按钮，不复用上一层状态；
- 最终结算显示通关/失败、Boss 击败数、战斗战绩、三连次数和最终阵容。

## 7. 自动化验收

EditMode 覆盖三层配置、跨层保留、Boss 奖励阶段、第三层通关、失败统计、奖励预留和动态高阶随从边界。PlayMode 覆盖切层 UI、选择面板重载以及三层胜利主链。现有 4A/4B 和商店、战斗测试必须全部保留。

Unity 人工通关、失败和 10-20 分钟时长验收按约定在阶段 2-4 代码全部完成后集中执行。

## 8. 实施结果

- 三层固定地图共 19 个可见节点，第一层 7 个、第二和第三层各 6 个。
- 第一、二层 Boss 奖励使用 `FloorComplete` 完成模式；第三层 Boss 胜利直接进入 `RunWon`。
- 跨层保留生命、阵容、酒馆、`RunTurn`、牌池、永久强化、延迟资源和 FIFO 奖励。
- Boss 动态随从候选受“当前酒馆等级 +1”和有限牌池双重约束。
- `RunStatistics` 已记录战斗、精英、Boss、三连和起止时间，`RunTest` 可显示切层与最终结果。
- 自动化结果：EditMode 73 / 73、PlayMode 11 / 11 全部通过。

代码与自动化完成标准已经满足；人工通关、失败和时长体验属于下一步集中验收。
