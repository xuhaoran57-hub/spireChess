# 随从设计文档

版本：0.1（当前实现基线）
关联文档：`game-design.md`、`development-plan.md`、`unity-project-prep-tech-plan.md`  
关联设计更新：`minion-pool-expansion-rebalance-design-v0.2.md`
状态：本文件保留阶段 5 的 50 个非 Token 基线说明；当前运行配置已按 v0.2 扩充为 64 个非 Token 和 3 个 Token，新增与再平衡内容以 `minion-pool-expansion-rebalance-design-v0.2.md` 为准。

## 1. 设计目标

本章节记录阶段 5 MVP 第一批 50 个非 Token 随从；v0.2 的 14 张新增随从、双尾狐影和原卡改造见关联设计更新。

结构：

- 5 个等级。
- 3 个主种族：铸魂、荒灵、星契。
- 1 个中立阵营：旅团。
- 每个等级每个主种族 3 个随从。
- 每个等级 1 个旅团随从。
- 每个等级共 10 个随从，总计 50 个。

### 1.1 v0.2 目标池（待实现）

- 三个主种族各等级调整为 `3 / 4 / 4 / 4 / 3`，每个主种族 18 个；
- 旅团调整为每等级 2 个，共 10 个；
- 合计 64 个非 Token 随从；新增双尾狐影后共有 3 个 Token，配置总数为 67；
- 相比当前版本新增 9 个主种族随从和 5 个旅团随从；
- v0.2 不让卡牌效果读取流派标签；路线名称只用于设计分析和固定构筑测试，不限制玩家混搭；
- 新卡设计、铸魂与荒灵原卡逐张改造、成长预算和验收门槛统一以 `minion-pool-expansion-rebalance-design-v0.2.md` 为准。

本文后续卡表仍用于描述当前 50 张池及已先行落地的少量改造，不在 v0.2 实现前将 14 张新卡混入当前配置口径。

设计原则：

- 关键词为全游戏通用，不做种族私有。
- 种族只体现关键词偏好和卡牌效果联动。
- 每个主种族至少提供两条流派方向。
- 旅团作为中立工具箱，不成为完整主流派。
- 数值暂按可测试版本设计，不代表最终平衡。

## 2. 通用关键词

MVP 初版优先使用较少关键词，降低学习成本。

| 关键词 | 说明 |
| --- | --- |
| 嘲讽 | 敌方优先攻击该随从。 |
| 护盾 | 抵挡一次伤害。 |
| 亡语 | 死亡时触发效果。 |
| 战吼 | 随从从备战区进入战斗区时触发；没有合法目标不阻止随从上场。 |
| 溅射 | 普通形态将攻击伤害拆分给目标左右两侧；金色形态对两侧各造成 100% 攻击伤害。 |

护盾规则：

- 护盾不能叠加。
- 已有护盾时再次获得护盾，不产生新的护盾层数。
- 只有从“无护盾”变成“有护盾”，才算“获得护盾”。
- 只有从“有护盾”变成“无护盾”，才算“失去护盾”。
- 护盾可以抵挡攻击、技能和溅射伤害。
- 移除护盾默认算作失去护盾，除非具体效果标注“不触发失去护盾效果”。

已确认：

- 购买随从进入备战区不触发战吼，只有从备战区进入战斗区时触发 `OnPlay`。
- 战斗区内部换位不触发；战斗区随从不能由玩家手动回收到备战区。
- 三连或明确卡牌效果使随从返回备战区后，再次上场会再次触发 `OnPlay`。
- 战吼没有合法目标时，随从仍正常上场并跳过无法结算的效果。
- 战吼只能选择操作完成后的最终战斗区；“友方随从”默认不包含战吼来源自身，需要作用于自己时必须显式使用 `Self`。
- 三连生成金色随从进入备战区时不触发；金色随从实际上场时执行 `goldenEffects` 中的 `OnPlay`。
- 三连生成的金色随从首次成功上场时额外赠送一张发现法术；该奖励每个金色实例只触发一次，不随重复上场重置。

金色继承规则：

- 三个材料的永久攻击和生命加成相加继承，永久关键词取并集继承。
- 金色随从保留配置自身的基础关键词，并叠加材料的永久关键词。
- 不继承护盾当前或已消耗状态、下一场战斗增益和各类触发计数。
- 备战区随从不能出售；普通和金色随从都必须先位于战斗区，才能在商店阶段出售。

待确认：

- 普通溅射的奇数伤害随机决定左右哪侧多 1 点；一侧为空时不转移该侧伤害。
- 护盾抵挡伤害后，是否允许原伤害的附带效果继续生效。

## 3. 属性变化规则

为了避免成长失控，同时保留构筑成型后的爽感，本游戏将属性变化分为永久强化和本场战斗临时强化。

### 3.1 默认规则

- 战斗阶段产生的属性变化，默认只持续到本场战斗结束。
- 商店阶段、奖励阶段、事件阶段、三连奖励产生的属性变化，默认永久保留。
- 若效果与默认规则不同，必须在文案中明确写出“本场战斗”或“永久”。
- Token 的属性变化默认只持续到本场战斗结束，因为 Token 战斗结束后会消失。

### 3.2 文案口径

临时强化使用以下写法：

- 本场战斗中，获得 +1/+1。
- 下一场战斗开始时，获得护盾。
- 下一场战斗开始时，获得 +2 攻击。
- 本场战斗中获得溅射。

永久强化使用以下写法：

- 永久获得 +1/+1。
- 使一个随从永久获得 +2 生命。
- 使所有友方随从永久获得 +2/+2。

### 3.3 当前设计口径

- 铸魂的失盾成长、补盾收益、反击收益，若未写“永久”，默认是本场战斗临时强化。
- 荒灵的死亡成长、召唤物死亡收益，若未写“永久”，默认是本场战斗临时强化。
- 星契在商店阶段由刷新、发现、使用法术带来的属性成长，若未写“下一场战斗”，默认永久保留。
- 星契的“下一场战斗开始时”获得护盾、攻击、溅射等效果，默认只持续到该场战斗结束。
- 旅团的金币、发现、出售收益属于经济效果，不涉及属性持续时间。

### 3.4 设计目的

- 普通战斗触发可以做得更爽，但不污染整局数值。
- 商店、法术、奖励和三连承担长期成长。
- 高等级金色随从可以突破普通规则，提供强力永久成长。

## 4. 种族与流派

| 种族 | 核心体验 | 流派 A | 流派 B |
| --- | --- | --- | --- |
| 铸魂 | 稳定防线、护盾、站位 | 护盾壁垒 | 失盾反击 |
| 荒灵 | 死亡收益、召唤、成长 | 频率召唤 | 亡语养大哥 |
| 星契 | 法术、刷新、发现、经济 | 法术回响 | 经济刷新 |
| 旅团 | 中立补位、经济、反制 | 过渡经济 | 反制补位 |

### 4.1 铸魂

主题：灵魂被封入铠甲、兵器和机关中的造物。  
偏好关键词：护盾、嘲讽、亡语。  
主要联动：获得护盾、失去护盾、相邻随从、受击后触发。

流派：

- 护盾壁垒：给自己或友方护盾，构建稳定防线。
- 失盾反击：护盾被打破后获得攻击力、反伤或其他收益。

### 4.2 荒灵

主题：山林、兽魂、藤蔓、古老生命力。  
偏好关键词：亡语、嘲讽。  
主要联动：友方死亡、荒灵死亡、召唤物、死亡后增益。

流派：

- 频率召唤：通过召唤物、召唤后立即攻击和死亡触发制造多次攻击。
- 亡语养大哥：通过死亡次数养成高身材核心随从。

### 4.3 星契

主题：观星者、符文术士、月相学派、命运契约。  
偏好关键词：战吼、护盾、溅射。  
主要联动：购买法术、使用法术、刷新商店、发现、出售随从、下一场战斗反馈。

流派：

- 法术回响：购买或使用法术后强化随从，并将本商店阶段的法术操作转化为下一场战斗的护盾、溅射或攻击提升。
- 经济刷新：通过免费刷新、出售收益和发现效果增加刷新次数，再让部分星契随从随刷新次数成长。

星契统计口径：

- “本商店阶段”指一次节点奖励或战斗后的商店操作阶段。
- “下一场战斗”指离开当前商店阶段后进入的下一次战斗。
- 免费刷新也计入刷新次数。
- 阶段 3 的“使用法术”只发生在同时承担阵容准备功能的商店阶段。
- 若效果引用本商店阶段的法术或刷新次数，进入下一场战斗后清空。

### 4.4 旅团

主题：商人、佣兵、医生、向导、赏金猎人。  
定位：中立工具箱。  
主要联动：金币、出售、购买、反制、临时补强。

流派：

- 过渡经济：帮助玩家平滑前期和转型。
- 反制补位：针对护盾、亡语、召唤、特定种族或高威胁敌人。

旅团规则口径：

- 旅团是中立阵营，不计入“主种族”数量判断。
- 主种族目前指铸魂、荒灵、星契。
- 若效果需要判断“场上数量最多主种族”，只统计玩家战斗区和备战区中的非旅团随从。
- 若多个主种族并列最多，后续实现时可以让玩家选择；MVP 阶段也可默认按铸魂、荒灵、星契的顺序取第一个。

## 10. Token 规则

Token 是战斗中临时生成的随从，只存在于当前战斗。Token 用于支撑荒灵的召唤、频率攻击和召唤物死亡联动。

### 10.1 当前 Token

| 名称 | 身材 | 关键词 | 来源 |
| --- | --- | --- | --- |
| 幼灵 | 1/1 | 无 | 幼鹿灵、双尾狐灵 |
| 迅捷幼灵 | 2/1 | 无 | 百鸣兽群，召唤后立即攻击一次 |

### 10.2 基础规则

- Token 拥有荒灵种族，但同时拥有召唤物标记。
- Token 占用战斗区格子。
- Token 可以被攻击、受到伤害、获得临时强化、获得临时护盾或临时关键词。
- Token 可以触发“召唤物死亡”相关效果。
- Token 不触发“非召唤物友方荒灵死亡”相关效果。
- Token 战斗结束后消失。
- Token 不进入备战区，不参与三连。
- Token 可以被本场战斗内的临时强化影响。
- Token 不能被购买、出售、发现或作为奖励获得。

### 10.3 永久强化规则

- Token 不能获得永久强化。
- 如果一个随机永久强化效果会命中 Token，则跳过 Token，从合法的非 Token 随从中选择。
- 如果没有合法的非 Token 随从，则该永久强化不生效。
- Token 战斗结束后消失，不会进入存档。

### 10.4 召唤位置

亡语召唤：

- 优先召唤到死亡随从原位置。
- 同一个亡语召唤多个 Token 时逐个结算：首个 Token 占死亡原位；若它具有立即攻击，则先完成该次攻击及其死亡连锁，再尝试生成下一个 Token。
- 前一个 Token 在立即攻击中死亡并释放格子时，后一个 Token 可以继续占用该格子；若前一个仍存活且没有其他空位，后一个才触发召唤失败补偿。
- 左右空位距离相同时，优先右侧，再选择左侧。
- 若没有空位，则召唤失败。

非亡语召唤：

- 优先召唤到最靠右的空位。
- 若没有空位，则召唤失败。

### 10.5 召唤失败

- 没有空位时，召唤失败。
- 召唤失败不算“召唤一个随从”。
- 召唤失败不会触发“每当你召唤一个随从”相关效果。
- 只有卡牌明确写了“若没有空位”或“未能召唤”时，才触发召唤失败补偿。

### 10.6 立即攻击

- “召唤后立即攻击一次”是额外攻击。
- 同一个单位因同一次召唤事件最多获得一次立即攻击；自身效果、友方随从效果或其他来源重复提供时不叠加，只生成一个立即攻击步骤。
- 去重只针对同一次召唤产生的立即攻击；战斗开始立即攻击、后续正常行动以及其他明确注明的独立攻击不受影响。
- 立即攻击不消耗该随从后续正常行动机会。
- 若召唤后没有合法目标，则不攻击。
- 立即攻击会触发攻击相关效果。
- 立即攻击造成的死亡正常触发亡语和死亡效果。
- 如果 Token 因立即攻击死亡，会触发“召唤物死亡”相关效果。

### 10.7 战斗结算顺序

- 普通行动按槽位从左到右，玩家与敌方交替执行；每个单位每轮最多执行一次普通行动。
- 本轮中新召唤的单位立即加入普通行动队列，在该阵营下一次普通行动时按当前位置参与排序，无需等待下一轮。
- 一次攻击先结算主目标、溅射和反击伤害，再统一移除死亡单位。
- 同一次攻击造成多个死亡时，先按敌方槽位从左到右处理亡语，再按攻击方槽位从左到右处理亡语。
- 多单位亡语按“召唤一个 → 汇总并去重该次召唤产生的立即攻击 → 结算该单位唯一一次立即攻击及死亡连锁 → 再召唤下一个”的顺序执行。
- 立即攻击生成独立结算步骤，不消耗普通行动；若单位仍存活，本轮仍可按当前位置执行一次普通行动。
- 护盾抵挡一次任意正数伤害，包括普通攻击、反击和溅射。

待确认：

- Token 死亡是否触发“友方随从死亡”这类不区分召唤物的效果。

## 11. 金色随从设计规则

金色随从是三连后的核心奖励，应明显强于普通随从，并允许玩家在流派成型后获得足够的成长爽感。

### 11.1 基础规则

- 三个相同普通随从合成为一个金色随从。
- 金色随从保留原随从的种族、等级、关键词和流派定位。
- 金色随从通常拥有更高属性和强化版效果。
- 三连后仍会赠送发现牌，发现规则另行设计。
- 金色随从的具体数值和效果可以单独配置，不强制使用简单翻倍。

### 11.2 金色属性与继承规则

金色随从基础属性采用“基础翻倍 + 继承永久加成”的规则。

计算公式：

```text
金色攻击 = 普通基础攻击 x 2 + 三个素材的永久攻击加成总和
金色生命 = 普通基础生命 x 2 + 三个素材的永久生命加成总和
```

其中：

```text
永久攻击加成 = 素材当前永久攻击 - 普通基础攻击
永久生命加成 = 素材当前永久生命 - 普通基础生命
```

示例：

```text
普通随从基础属性：2/3

素材 A：2/3，永久加成 0/0
素材 B：3/4，永久加成 +1/+1
素材 C：2/5，永久加成 +0/+2

金色基础属性：4/6
继承永久加成：+1/+3
最终金色属性：5/9
```

继承边界：

- 战斗临时属性不继承。
- 护盾状态继承。
- 战斗中获得的临时关键词不继承。
- 商店阶段获得的永久关键词继承。
- Token 不能参与三连。
- 金色随从后续继续获得永久加成时，直接加在金色当前永久属性上。

### 11.3 金色效果设计原则

- 1-2 级金色随从主要强化即时战力或经济收益，不建议提供强永久成长。
- 3 级金色随从可以提供少量永久成长，但需要谨慎。
- 4-5 级金色随从可以作为流派大成核心，允许提供强永久成长。
- 低频、一次性、难触发的金色效果可以大胆设计为永久强化。
- 高频、可循环、容易刷次数的金色效果需要谨慎，尤其是召唤物死亡、刷新商店、失去护盾等高频触发。
- 若金色效果会产生永久强化，文案必须明确写“永久”。

### 11.4 永久成长边界

推荐允许：

- 普通铸魂、荒灵随从在每场第一次或前两次核心机制触发时获得少量永久成长，后续触发仅提供本场强化。
- 高等级金色随从的亡语提供全队永久强化。
- 高等级金色随从在满足流派条件后提供少量永久成长。
- 金色随从强化相邻随从或同种族随从，体现成型阵容收益。
- 金色随从提高永久成长次数或数值，但不同时无限提高次数和数值。

需要谨慎：

- 每次 Token 死亡都能无上限触发的永久成长。
- 每次刷新商店带来的永久成长。
- 每次失去护盾带来的永久成长。
- 每次使用法术带来的全队永久成长。

### 11.5 示例方向

以下示例按 v0.2 目标口径描述；当前运行配置与目标设计的差异见扩池再平衡文档。

铸魂示例：

- 普通千环守墓者：亡语使所有友方随从本场战斗获得 +2/+2，并使两个存活的非 Token 铸魂永久获得 +1/+1。
- 金色千环守墓者：亡语使所有非 Token 友方随从永久获得 +2/+2。

荒灵示例：

- 普通群山古灵：每场前 3 次非 Token 友方荒灵死亡时，使所有存活的非 Token 荒灵永久获得 +1/+1。
- 金色群山古灵：每场前 3 次非 Token 友方荒灵死亡时，使所有存活的非 Token 荒灵永久获得 +2/+2。

星契示例：

- 普通陨星先知：每个商店阶段前 2 次使用法术后，所有星契永久获得 +1/+1。
- 金色陨星先知：每个商店阶段前 2 次使用法术后，所有友方随从永久获得 +1/+1。

旅团示例：

- 普通旧塔向导：发现一个与你场上数量最多主种族相同的随从，等级不高于当前酒馆等级。
- 金色旧塔向导：发现一个与你场上数量最多主种族相同的金色候选奖励或高等级随从，具体规则待定。

### 11.6 设计目标

- 让玩家凑齐高等级三连后获得明显回报。
- 让大成流派可以碾压普通战斗，但 Boss 和精英仍可通过机制形成挑战。
- 让好阵容和差阵容在后期有足够区分度，便于设计敌人强度曲线。
- 通过“普通版受限永久成长 + 金色版加速但封顶”补足铸魂和荒灵的长期属性成长来源。

## 13. 待调整问题

本版本已完成 MVP 基线，v0.2 后续重点验证以下问题：

- 从 50 张扩充到 64 张后，核心获取率和三连率是否过度下降。
- 铸魂护盾链是否过强。
- 荒灵召唤物在 5 格战斗区里是否容易卡格。
- 铸魂与荒灵的普通版受限永久成长，是否能缩小正常发育和最高发育的差距。
- 金色群山古灵与金色终花吞世者是否仍会形成乘算滚雪球。
- 星契刷新和法术收益是否会导致经济流派过强。
- 旅团是否只是工具牌，还是会变成最优中立流派。
- 主种族、等级范围和复制目标等由玩家主动选择的定向奖励，能否抵消扩池稀释，同时不额外放大经济刷新。
- 新旧卡效果所需的白名单条件与动作，能否继续由现有效果系统承载而不引入任意脚本 DSL。

## 14. 下一步

建议后续按以下顺序继续：

1. 先实现铸魂、荒灵原有随从的 v0.2 改造和金色终结器上限。
2. 在当前 50 张池中重跑六套固定构筑，校准正常与最高成长预算。
3. 加入 9 张主种族新卡，再加入 5 张旅团新卡及百技学徒的相邻关键词复制。
4. 同步发布清单、奖励、遭遇和内容校验。
5. 运行扩池后的固定构筑、三连率、完整三层和 Unity 人工验收。
