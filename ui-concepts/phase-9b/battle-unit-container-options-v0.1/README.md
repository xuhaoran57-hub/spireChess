# 无关键词战斗单位容器比较 v0.1

三套方案均使用内置 GPT Image 工作流生成，并放入同一正式战斗界面。共同约束：

- 五个玩家战斗槽位全部填满，用于比较整排密度。
- 只显示角色立绘、攻击和生命。
- 不显示名称、等级、种族、说明或关键词。
- 不加入护盾、嘲讽、亡语、溅射等状态效果。
- 完整卡牌预留给点击或鼠标悬停浮层。

## A：拱顶旅团肖像立牌

文件：`option-a-arched-traveling-standee.png`

- 角色立绘面积最大，竖向轮廓稳定。
- 攻击、生命与角色主体的视觉关系清楚。
- 外围和卡牌背后留有较稳定的状态效果空间。

## B：圆形绘本徽章

文件：`option-b-circular-storybook-medallion.png`

- 头像识别最快，整排节奏最整齐。
- 对高大、长武器或横向动作的角色裁切较强。
- 圆周适合护盾，但嘲讽、亡语等状态容易争抢同一圈层。

## C：不规则羊皮纸立像

文件：`option-c-irregular-parchment-figure.png`

- 旅团绘本风格最强，角色像可移动的绘本纸偶。
- 不同角色可以拥有更自由的外轮廓。
- 五个单位并排时边缘噪声较多，状态效果需要更严格地控制范围。

## 初步建议

优先继续验证 A。它在角色面积、队列整齐度和状态效果扩展空间之间最均衡。C 可以作为召唤物、衍生物或特殊旅团单位的差异化容器；B 更适合头像、行动顺序或目标选择界面。
