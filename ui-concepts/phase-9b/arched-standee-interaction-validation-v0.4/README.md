# 拱顶旅团肖像立牌交互验证 v0.4

本组验证稿使用内置 GPT Image 工作流生成，沿用已确认的拱顶肖像立牌和关键词固定区域。

## 01：鼠标悬停

- 战斗区包含五个不同角色与身材的立牌。
- 立牌本体只显示立绘、攻击和生命。
- 悬停单位使用米白暖光轮廓和轻微抬升。
- 完整卡牌在单位上方偏右出现，并用短羊皮纸指针关联。
- 弹出卡不遮挡悬停单位、相邻目标、战斗日志和顶栏。

## 02：点击锁定

- 点击后的完整卡牌由黄铜固定点和短系带标记为锁定状态。
- 右侧栏上部切换为单位详情，下部保留战斗日志。
- 详情面板承载完整属性、关键词和效果说明。
- 战场仍可继续观察，不使用全屏遮罩。

## 03：目标选择

- 玩家攻击者使用暖金轮廓和抬升底座。
- 敌方也统一改为拱顶肖像立牌。
- 嘲讽单位仅使用加强底座，并通过红金手绘目标带捕获攻击指向。
- 普通可选目标使用细米白轮廓。
- 不可选目标去色，禁止标记只落在单位地面阴影处。

## 建议冻结

- Hover 延迟：150–200ms。
- 移出关闭延迟：约 100ms。
- 点击锁定后，只有再次点击、点击空白处或单位离场才关闭。
- 悬浮卡优先放在单位上方；空间不足时向左右翻转，禁止覆盖右侧战斗日志。
- 战斗单位、悬浮完整卡和详情面板使用三个独立 UI 层。
