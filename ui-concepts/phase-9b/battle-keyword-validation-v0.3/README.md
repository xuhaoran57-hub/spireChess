# 战斗关键词运行时验证 v0.3

## 本轮目标

将生成式书签、护盾、亡语与溅射原件做减法处理，并在真实 160×240 卡牌及现有正式战斗界面中验证，不再使用程序绘制的几何替代物。

## 减法规则

- 四枚书签裁掉透明留白，统一限制在约 31×48 像素；仅保留材质主色、外轮廓和核心符号。
- 书签最多显示三枚；第四个及以后改为 `+N` 页签，完整关键词只在详情面板显示。
- 护盾缩到卡框内沿，并将效果亮度降到原图的 38%，避免遮住名称、说明和攻防。
- 亡语采用“蜡封崩开”关键姿势，只覆盖死亡单位的立绘区域。
- 溅射采用“主命中分向两个次目标”关键姿势，缩小实体书签，支线跨越三个实际战斗槽位。

## 输出

- `runtime-assets/`：四枚运行时尺寸透明书签。
- `renders/battle-keywords-runtime-1920x1080.png`：单、双、三关键词、三关键词＋护盾、四关键词溢出。
- `renders/battle-deathrattle-runtime-1920x1080.png`：亡语事件置于正式战斗界面。
- `renders/battle-splash-runtime-1920x1080.png`：溅射事件置于正式战斗界面。

## 结论

- 书签的材质语言在 160×240 下成立，但核心符号已经接近可读下限，不应继续缩小。
- 护盾完整态仍应优先强调外轮廓；内部水彩填充只作轻微染色。
- 亡语事件适合绑定死亡单位立绘，不应扩散到整张卡。
- 溅射图集可提供书签展开与墨迹材质，但目标连线必须由运行时按实际目标坐标生成。

## 复现

运行 `python tools/compose_battle_keyword_runtime_v03.py`。
