# 拱顶旅团肖像立牌运行资产 v0.5

本批资产通过内置 GPT Image 工作流分别生成，并从整屏验证稿拆分为可独立挂载的运行部件。

## 目录

- `sources/`：原始色键背景生成图。
- `alpha/`：高分辨率透明母版。
- `runtime/`：裁边并缩放后的 2× 运行尺寸资产。

## 运行资产

| 文件 | 尺寸 | 用途 |
|---|---:|---|
| `standee-frame.png` | 199×376 | 空拱顶立牌框 |

由于保持原始比例，部分文件没有填满目标包围盒；在 Unity 中应按锚点定位，不要再次非等比拉伸。

## 参考布局

- 逻辑立牌尺寸：约 148×188。
- 立牌框：中心锚定。
- 角色立绘：先按拱顶内部区域裁切，再置于框体下层。
- 攻击圆章：左下；生命圆章：右下。
- 亡语：顶部中心，位于框体前层。
- 溅射：攻击圆章上方，位于数值层前方。
- 嘲讽：底部中心，位于立牌框和数值圆章下层。
- 护盾：覆盖肖像与框体表面，但应位于文字和交互高亮层下方。

## 混合与文本

- 除护盾外均使用普通 Alpha 混合。
- 护盾使用 `Screen` 或 `Additive`；黑色背景不显示。
- 攻击与生命数字不得烘焙进图片。
- Hover、选中、目标线和不可选状态不属于关键词资产，应由独立交互层实现。

## 复现

透明抠图使用 imagegen skill 的 `remove_chroma_key.py`。运行尺寸资产由 `python tools/prepare_arch_standee_runtime_assets.py` 生成。
