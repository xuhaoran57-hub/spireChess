# 旅团绘本战斗关键词与状态表现验证 v0.1

## 冻结规则

- 商店商品与手牌不额外展示关键词，完整说明进入详情面板。
- 商店阶段上阵区显示静态战术关键词和当前状态。
- 正式战斗界面显示关键词徽章、整卡状态覆盖及事件动画。
- 嘲讽、亡语、溅射使用稳定的大号绘本徽章。
- 护盾使用冷蓝水彩整卡覆盖；护盾消失后覆盖必须完全移除。
- 战吼仅在入场结算时短暂出现，不常驻。
- 永久成长不设专属图标，通过文案、属性变化和短暂“永久”浮签表达。

## 现有代码映射

| 表现 | 现有数据或事件 |
| --- | --- |
| 当前护盾 | `BattleCardViewModel.HasShield` |
| 嘲讽/亡语/溅射 | `BattleCardViewModel.Keywords`；`Cleave` 需补中文映射 |
| 护盾获得/失去 | `BattlePlaybackEventKind.ShieldGained` / `ShieldLost` |
| 护盾抵挡 | `DamageApplied.WasBlocked` |
| 亡语触发点亮 | 当前可由 `UnitDied` 与播放队列阶段驱动；若需精确到每次额外亡语，G2 再评估专用事件 |
| 溅射目标线 | `BattlePlaybackEvent.SplashTargetIndexes` |
| 跳过/2× | 复用现有播放控制，不改变领域结算 |

## 评审重点

1. 160×240 战斗卡上的徽章能否在 1920×1080 下立即辨认。
2. 嘲讽与护盾叠加是否仍然清晰。
3. 护盾破裂后玩家能否确认状态已经消失。
4. 亡语触发期间卡牌是否保留到效果完成。
5. 溅射预告是否明确主目标与相邻目标。
6. 所有表现是否不会遮挡等级、攻防和目标框。
