# 战斗关键词空间语言验证稿 v0.4

## 目的

在重新生成美术效果前，先冻结四种关键词与 160×240 战斗卡的空间关系。本稿不代表最终材质与动画质量。

正式视觉验证稿使用内置 GPT Image 工作流生成；程序绘制版本仅保留为空间构图参考，不作为视觉评审依据。

## 冻结的空间语言

- 护盾：卡牌表面层。低透明度染色与内沿轮廓，不越过名称和攻防信息。
- 嘲讽：卡牌外框层。左右护卫结构与顶部压场轮廓，不覆盖立绘与文字。
- 亡语：卡牌背后层。紫色封印环从卡底和两侧露出，触发时才上浮至立绘区。
- 溅射：攻击关系层。静默态只在立绘区保留短小分叉轨迹；攻击时才延伸到实际目标。

## 多关键词规则

- 四个关键词可以同时存在，因为它们不占用同一空间层。
- 静默态只承担识别，不同时播放完整动画。
- 某关键词激活时，其他常驻效果降到约 30%亮度。
- 名称横条、攻击与生命数值是永久禁入区。

## 验证结论

- 书签可以移除，不再需要独立关键词栏。
- 护盾、嘲讽和亡语在静默态下已有清楚的空间区分。
- 溅射不适合形成大面积常驻覆盖，必须保持为立绘区短标记与攻击时目标连线。
- 下一阶段可以基于这四个冻结剪影，分别生成旅团绘本材质，不让生成模型重新决定布局。
