# 随从池扩充与原卡再平衡设计 v0.2

版本：0.2 设计稿
状态：设计已确认方向，代码与运行时配置待实现
基线：阶段 5 当前 50 个非 Token 随从、2 个 Token

## 1. 目的

本轮同时完成两件事：

1. 将三个主种族每级随从数调整为 `3 / 4 / 4 / 4 / 3`，旅团调整为每级 2 个；
2. 在新增随从之外改造铸魂、荒灵原有随从，使四个非星契流派获得从低级核心开始的受限永久成长。

星契的法术回响和经济刷新保持当前强度，继续作为稳定成长基准。铸魂和荒灵采用“双层成长”：

- 商店外永久面板约达到星契基准的 65%-80%；
- 完成护盾、召唤或死亡触发后，永久成长与战斗临时成长之和达到星契的 100%-115%；
- 护盾、立即攻击、亡语和溅射作为额外流派价值，不再用无限永久成长补偿。

## 3. 成长预算与通用规则

### 3.1 第 10 回合正常目标

| 流派 | 永久面板目标 | 完成核心触发后目标 | 额外价值 |
| --- | --- | --- | --- |
| 护盾壁垒 | 约 40/50 | 约 50/70 | 5-8 次护盾抵挡 |
| 失盾反击 | 约 45/35 | 约 75/40 | 护盾与反击爆发 |
| 频率召唤 | 非 Token 约 40/40 | 含 Token 约 65/60 | 立即攻击与溅射 |
| 亡语成长 | 约 40/50 | 死亡触发后约 65/70 | 亡语与残局核心 |
| 法术回响 | 当前约 65/66 | 保持当前基准 | 稳定、无需敌方配合 |
| 经济刷新 | 当前约 48/60 | 保持当前基准 | 经济与下一战护盾 |

六套构筑只是平衡测试样本，不是系统预设流派。玩家可以自由混合种族、关键词和成长组件，任何卡牌效果都不得依赖“属于某个流派”才能生效。

### 3.2 永久成长模板

- 普通版本：每场第一次核心机制触发提供少量永久成长，后续触发只提供战斗临时成长；
- 金色版本：将永久次数提高到 2 次或将数值翻倍，不同时无限提高次数和数值；
- 普通卡每场永久收益通常不超过 2-4 点总属性；
- 单张金色卡每场永久收益通常不超过普通版的 2 倍；
- 优先使用机制触发，不用“仅胜利后成长”作为主要入口，避免胜者滚雪球；
- 战斗永久差量继续只回写有 `SourceInstanceId` 的非 Token 持有实例。

旅团仍只是中立工具箱，不形成需要追齐的固定套系。新增牌分别承担保血、前排、站位与关键词拼接、召唤反制和单核反制，不能稳定提供比同等级主种族核心更高的纯身材。

百技学徒不替玩家寻找预设卡组，而是把玩家已经拥有的战斗关键词转换成站位收益。它可以跨种族组合，只复制触发时实际存在的嘲讽、护盾和溅射，不复制身材、战吼、亡语或其他效果文本；重复关键词不叠加。

## 8. 扩池稀释与获取补偿

随从类型从 50 增加到 64，扩大 28%。若商店规则不变：

- T3 累计类型从 30 增加到 39，指定核心命中率约下降 23%；
- T5 累计类型从 50 增加到 64，指定核心命中率约下降 22%；
- 三连和双核心构筑会明显变慢。

第一轮不增加商店格子，避免额外加强经济刷新。采用以下补偿：

1. 旧塔向导继续提供“场上数量最多主种族”的发现，作为旅团现有的种族定向发现入口；
2. 精英/Boss 奖励可以提供玩家主动选择的主种族或等级范围筛选，不读取预设流派；
3. 复制雏形继续帮助玩家围绕自己已经选中的 1-2 级核心完成三连；
4. 每层至少有一次由玩家主动选择主种族、等级范围或复制目标的定向机会，但不保证具体卡；
5. 各等级有限牌池副本数第一轮保持不变，根据实测三连率再调整。

## 9. 技术需求

多数效果可复用现有 `trigger + condition + target + value + limit`。以下能力需要实现前确认或扩展：

- 百技学徒在战斗开始时读取相邻单位当前的嘲讽、护盾和溅射，并仅在本场复制；不复制属性或效果文本，重复关键词不叠加，没有合法相邻单位或没有可复制关键词时不产生效果；
- 双尾狐影需要支持 Token 自身携带亡语并继续生成 Token；狐影和幼灵必须沿用“召唤一个、完成立即攻击与死亡连锁、再召唤下一个”的顺序；
- 双尾狐影必须始终带召唤物标记且不带持有实例来源，死亡时不得触发非 Token 死亡条件，战斗结束后不得回写任何永久差量；
- 每次成功召唤需要产生可追踪的召唤事件标识；按“召唤事件 + 被召唤单位实例”对 `ImmediateAttack` 请求去重，多个来源只允许一个请求进入攻击队列；
- 敌方 Token 召唤观察；
- “第 N 次失盾/召唤/死亡后”条件；
- 下一场战斗属性修改的多目标选择；
- 按敌方最高攻击计算且带数值上限；
- 终花从永久吸收当前攻击改为固定永久成长、当前攻击仅作战斗临时值；
- 多个永久观察者同时存在时的事件顺序和每场使用次数隔离。

实现时不得引入任意脚本 DSL；只为本设计中已经出现的条件和动作增加白名单能力。

v0.2 不让任何卡牌、发现或奖励效果读取 `archetypes`。现有字段只属于旧版分析元数据，当前也未参与运行时效果；实现 v0.2 时停止给新卡配置该字段，并逐步从卡牌配置、校验和遥测中移除。文档中的“护盾壁垒”“频率召唤”等名称只保留为平衡分析用语。

## 10. 验收门槛

### 10.1 内容与配置

- 非 Token 随从恰好 64 个，Token 为 3 个，总随从配置数为 67；
- 主种族各等级数量严格为 `3 / 4 / 4 / 4 / 3`；
- 旅团各等级严格为 2 个；
- 从设计功能上检查 T2-T4 每个主种族的两条主要机制路线各有约 2 张支撑卡，但不新增流派标签或按流派校验配置；
- 百技学徒只复制相邻单位在战斗开始时实际拥有的嘲讽、护盾和溅射，复制结果不会永久回写；
- 普通/金色不熄炉王每场转移护盾上限分别为 3 次和 6 次；
- 狐群巢母只召唤一个双尾狐影；狐影死亡后两个幼灵逐个结算，并且整条召唤链都不能触发非 Token 荒灵死亡效果；
- 迅捷幼灵与万蹄奔潮同时提供立即攻击时，该幼灵只立即攻击一次，之后若仍存活仍可进行本轮正常攻击；
- 所有普通/金色文案与效果配置一致；
- 发布清单、遭遇和奖励池同步收录新内容。

### 10.2 正常构筑

- 六套固定构筑第 10 回合永久面板符合第 3 节目标区间；
- 非星契四套完成核心触发后的总身材不低于经济刷新基准；
- 法术回响仍可以是最稳定构筑，但第一轮输出不应领先第二名超过 35%；
- 频率召唤在标准压力阵容下至少有一个非 Token 核心存活，或以足够高的 Token 输出换取全灭；
- 四套非星契构筑在失败战斗中仍能通过机制触发获得有限永久成长。

### 10.3 最高构筑

- 最高发育与正常发育的第一轮伤害比尽量不超过 2.5；
- 任一固定构筑第一轮标准木桩伤害不超过 400；
- 金色群山与金色终花组合不再出现永久成长乘算；
- 金色版本提升速度或上限，但不成为流派唯一永久成长入口。

### 10.4 扩池体验

- 在不增加商店格子的情况下，三层单局至少能稳定形成一套双核心构筑；
- 至少 20 局记录核心首次获得回合、三连次数、定向发现使用次数；
- 若完整单局三连中位数低于 1 次，再考虑提高低级副本数或增加定向复制奖励。

## 11. 建议实施顺序

1. 先实现原有铸魂/荒灵卡牌改造和金色终结器上限；
2. 用现有 50 张池重跑六套固定构筑，确认成长预算；
3. 加入 9 张主种族新卡和双尾狐影 Token，并完成嵌套亡语、格子占用、同次召唤立即攻击去重及正常行动组合测试；
4. 加入 5 张旅团新卡和百技学徒的相邻关键词复制；
5. 更新发布清单、奖励、遭遇和内容校验；
6. 跑扩池后的固定构筑、三连率和完整三层测试；
7. 再决定是否调整牌池副本数，不先增加商店格子。
