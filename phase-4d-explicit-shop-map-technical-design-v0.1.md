# 阶段 4D 显式商店地图技术方案 v0.1

日期：2026-07-19
状态：已实现并完成自动化验证；Unity 显式商店地图、完整三层单局和 Boss 失败重试人工验收已通过

## 1. 目标

将商店从战斗节点的隐式前置阶段拆分为地图上的独立节点。每层任意完整路线固定经过 5 个商店和 5 场战斗，第 5 场为 Boss；路线分支不再通过额外商店次数制造经济差异。

该改造同时修复 Boss 失败重试会增加 `RunTurn`、发放新金币、补货、推进升级降价并重复触发商店效果的问题。

## 2. 单层拓扑

```text
S0 初始商店
  → C1 固定普通战斗
  → S1 商店
  → C2A 精英 / C2B 普通
  → U 锻造 / 事件 / 恢复
  → S2 商店
  → C3 固定机制战斗
  → S3 商店
  → C4A / C4B 后期战斗
  → S4 Boss 前商店
  → C5 Boss
```

`MapValidator` 对每条起点到 Boss 的路径忽略功能节点后验证严格交替序列：

`Shop → Combat → Shop → Combat → Shop → Combat → Shop → Combat → Shop → Boss`

三层地图均包含 15 个可见节点；任意实际路线访问 11 个节点，其中 5 个商店、5 个战斗和 1 个功能节点。

## 3. 经济和状态语义

- `ShopTurn` 只在进入显式 `Shop` 节点时增加，继续驱动基础金币、升级降价和第 10 商店回合快照。
- `RunTurn` 作为 `ShopTurn` 的兼容别名保留，避免破坏既有统计和离线聚合格式。
- `MapStep` 在首次进入地图节点时增加，用于流程和遥测；Boss 重试不增加 `MapStep`。
- 战斗、事件、锻造和恢复不推进商店经济。
- 延迟金币、免费刷新和升级折扣只在下一个真实商店节点消费。
- 商店节点结束后返回地图；战斗节点进入后直接生成战斗快照并进入 `Battle`。
- 第一、二层 Boss 奖励在下一层起始商店消费。

## 4. Boss 重试

- Boss 失败照常结算生命伤害和尝试统计，Boss 节点保持 `Current`。
- 存活时重试直接创建新的战斗尝试，不进入商店、不增加 `ShopTurn` 或 `MapStep`，金币保持 0。
- 同一个地图节点使用由单局种子和节点 ID 派生的固定战斗种子；不改变阵容/站位时不会通过重试刷新战斗随机流。
- 未击败 Boss 的尝试不提交永久差量、繁茂、战后卡牌奖励或核心激活证据；击败 Boss 的尝试才提交成长。

## 5. 遭遇曲线

原始总身材口径为开战前敌方攻击/生命总和，包含金色基础值和槽位加成，不包含护盾、召唤物、战斗开始效果和战斗内成长。

| 楼层 | C1 | C2 普通 / 精英 | C3 | C4 | C5 Boss |
| --- | ---: | ---: | ---: | ---: | ---: |
| 1 | 1/2 | 4/5 或 4/7 | 5/9 | 8/12 或 7/11 | 11/23 |
| 2 | 14/18 | 18/22 或 20/26 | 23/27 | 27/29 或 25/30 | 36/40 |
| 3 | 62/68 | 70/76 或 76/82 | 80/86 | 90/94 | 100/100 |

正式地图使用 21 个遭遇配置。原始身材由 EditMode 测试逐项锁定；效果强度与实际胜率仍需在新完整单局样本中校准。

难度修订（2026-07-19）：根据首次完整单局反馈，将第一、二层从“平衡候选压力测试”调整为“构筑成型引导”。除降低槽位加成外，第一层后期普通战减少一个敌方槽位，第二层普通路线减少早期满编压力并移除后期大部分金色单位；第一、二层精英与 Boss 的失败伤害加成同步减半。第三层阵容、金色状态、槽位加成与 62/68 → 100/100 曲线保持不变。

## 6. UI 与遥测

- `RunTest` 显示商店节点、战斗序号、`ShopTurn`、`MapStep` 和每层 5 战节奏。
- 11 列地图使用自适应紧凑按钮宽度，避免固定 220px 节点重叠。
- 商店结束按钮在地图流程中显示“离开商店”，成功后返回 `RunTest`；独立商店测试仍保留“进入战斗”语义。
- 己方普通卡继续使用银灰边框；金色卡使用高对比金边、金色卡名和常驻“金色”徽章，紧凑模式不再只依赖边框色差。
- 战斗卡的攻击/生命组合文本在固定 76×32 逻辑区域内按实际字体宽度自适应字号，三位数及 `999/1200` 可完整显示，短数值保持原 20 号字号。
- 遥测在 `RunStarted`、`NodeEntered`、`ShopEvent`、`Turn10Snapshot` 和 `RunEnded` 中同时记录兼容 `runTurn`、`shopTurn` 和 `mapStep`。

## 7. 自动化结果

- EditMode：185 / 185
- PlayMode：18 / 18
- 地图配置：三层均为 5 商店/5 战斗路径，Boss 唯一且为第 5 战。
- 遭遇配置：21 个正式遭遇全部命中预设原始身材，终局 Boss 为 100/100。
- Boss 重试：不增加商店回合、地图步数或金币，战斗种子保持一致。
- 随从发现：牌池预留前排除来源随从 ID；命运洗牌师刷新 3 次后的同级星契发现只提供另外两张候选，不再形成自发现—出售循环。

本改造将内容版本升级为 `5.2.0`；前两层首次难度修订升级为 `5.2.1`；随从发现自循环与金色识别修复进一步升级为 `5.2.2`。这些版本不得与旧 `5.1.0` 的人工完整单局直接合并，`5.2.2` 也不与未完成验收的 `5.2.0`、`5.2.1` 样本合并。旧 R16 freeze、旧 S1 和种子 2000 原始记录保留为改造前证据；新候选提交后重新运行 S1，并从种子 2000 重启 S2 人工采集。
