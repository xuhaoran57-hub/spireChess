## 3. 建议的 G1 成本门槛

在没有独立人工计时和逐次账单的情况下，建议 G1 先按可复核的生产效率签字：

| 指标 | 建议门槛 | 当前证据 | 结论 |
| --- | ---: | ---: | --- |
| 风格冻结后的图像调用 | 平均 ≤ 2 次/最终资产 | 6 张随从与 4 组件均为 1 次 | 达标 |
| 风格冻结后的评审包墙钟 | ≤ 10 分钟/最终资产 | 约 4:19/资产 | 达标 |
| 全研发链平均调用 | ≤ 4 次/最终资产 | 锚点 3.25；组件 1.50；总体 2.11 | 达标 |
| 单个高风险英雄资产调用上限 | ≤ 8 次 | 不熄炉王最高 7 次 | 达标 |
| 当前 v0.2 锚点排期 | ≤ 30 分钟/最终资产 | 约 16:39/张 | 达标 |
| 稳态批次系统性返工 | 不允许出现新的共同风格或种族规则推翻 | 6 张随从、4 组件均未推翻冻结方向 | 达标 |

推荐结论：

> G1 的“是否具备可接受成本批量生产”可以按生产效率门槛判定为通过。当前证据只
> 支持风格与离线资产生产，不包含 G2 Unity 接入/验收成本；后者必须在 G2 逐项记录，
> 再用于 9C 全量产能估算。

项目方仍需明确选择：

- 接受上述生产效率门槛；或
- 填写自定义的人民币上限、人工费率和图像调用计费口径。

   [OpenAI Services Agreement](https://openai.com/policies/services-agreement/)。

两类官方协议的共同核心是：

- 在 OpenAI 与用户之间、并在适用法律允许范围内，用户保留 Input 权利并拥有
  Output；OpenAI 将其可能拥有的 Output 权利转让给用户；
- 用户必须拥有向服务提供所有 Input 所需的权利、许可和权限；
- Output 可能不唯一，其他用户可能收到相似内容；
- 用户负责人工评估 Output 是否适合目标用途；
- 不得把 AI Output 表述为完全由人类生成；
- 使用仍须遵守适用法律、OpenAI Terms/Agreement 与
  [Usage Policies](https://openai.com/policies/usage-policies/)。

因此，“OpenAI 允许用户拥有 Output”并不自动证明：

- 参考图中的每个第三方元素都已获授权；
- Output 在目标法域一定获得著作权保护；
- Output 不会与第三方作品、商标、角色或其他用户的 Output 相似；
- OpenAI 为个人账号的每个商业发行场景提供不侵权保证。
