## 1. 目标

增加一套单局内永久生效的被动能力系统，正式名称为“遗珍”。遗珍不进入备战区、不占随从牌池中的卡牌位置，也不作为可主动使用的消耗品。

遗珍的主要作用是：

- 让第一、二层 Boss 胜利产生能改变后续构筑方式的确定性成长节点；
- 为事件节点增加“支付生命换取长期能力”的高风险选项；
- 在不继续扩充卡牌数量的情况下，提高单局间差异；
- 为完整地图中的 Boss、事件和路线收益提供新的长期奖励维度。

## 2. v0.1 边界

本版本包含：

- 8 件冠冕级遗珍，由第一、二层 Boss 掉落；
- 7 件奇物级遗珍，由事件获得；
- 无持有数量硬上限、同名遗珍不重复的动态持有列表；
- Boss 强制三选一、事件支付生命三选一；
- 商店、战斗、奖励队列和生命系统所需的被动规则钩子；
- 配置校验、确定性随机、遥测和纯 C# 自动化测试方案。

本版本不包含：

- 局外收藏、图鉴解锁或跨局成长；
- 遗珍出售、升级、合成和主动使用；
- 同一遗珍重复获得或层数叠加；
- 第三层最终 Boss 后无后续消费场景的局内遗珍奖励；
- 正式图标、美术、音效和复杂获得动画；
- 使用通用脚本 DSL 表达任意遗珍逻辑。

## 3. 等级、持有与获取

### 3.1 等级

| 等级 | 配置值 | 来源 | 强度定位 |
| --- | --- | --- | --- |
| 冠冕级 | `Crown` | 第一、二层 Boss | 改变构筑规则或提供高额长期收益 |
| 奇物级 | `Curio` | 特定事件 | 温和经济、续航、节奏或单目标战斗收益 |

等级既是获取来源，也是 UI 外框、候选池和校验规则。v0.1 不在同一来源内继续划分普通、稀有或传说。

### 3.2 持有规则

- 单局不设置遗珍持有数量硬上限。
- 已拥有的遗珍从所有后续候选中排除，同名遗珍不能重复获得或叠层。
- 第一、二层 Boss 在当前三层流程中自然提供最多 2 件冠冕级遗珍，但这不是系统上限；未来增加额外 Boss 时无需改数据结构。
- 每次遗珍事件最多获得 1 件奇物级遗珍；同局可以通过多个事件继续获得其他奇物级遗珍。
- 当前固定地图每层只有一个事件节点，因此即使三层都命中遗珍事件，现有单局的实际最大值也只是 2 件冠冕级加 3 件奇物级，共 5 件；这是内容结构形成的自然上限，不写入领域规则。
- 当某一等级已没有未持有候选时，对应掉落或事件选项不再生成。
- 因为没有硬上限，v0.1 不需要替换和丢弃流程；强度由 Boss 数量、遗珍事件密度、生命代价和候选池共同控制。

### 3.3 Boss 获取

第一、二层 Boss 首次胜利后：

1. 只在成功结算的 Boss 尝试上生成一次候选；失败重试不生成候选。
2. 从未持有的冠冕级遗珍中物化 3 件，优先使用不同类别。
3. 进入阻塞式三选一，不允许跳过。
4. 选择后完成 Boss 节点并进入 `FloorComplete`。

冠冕级遗珍替换现有第一、二层 Boss 的普通功能奖励，不再额外叠加原有随从、法术、金币、升级折扣或永久属性三选一。否则新系统会在没有重新标定遭遇曲线前造成双重奖励膨胀。

第三层 Boss 是当前单局终点，v0.1 不生成无法在本局消费的遗珍。未来若增加无尽模式或局外收藏，再为最终 Boss 增加结算型掉落。

### 3.4 事件获取

事件池可以生成“密封秘库”类事件，只要仍有未持有的奇物级遗珍即可：

1. 玩家选择查看遗珍，系统物化 3 件未持有的奇物级候选。
2. 候选物化后仍允许离开，不扣生命。
3. 选择具体遗珍时，在同一事务内扣除生命并获得遗珍。
4. 生命代价必须保留至少 1 点生命；不满足时所有候选均不可提交。

按获得时剩余可消费节点递减生命代价：

| 楼层 | 生命代价 |
| --- | ---: |
| 第一层 | 5 |
| 第二层 | 4 |
| 第三层 | 3 |

不能先扣生命再等待另一个可取消的选择。遗珍选择、生命校验、扣血、获得和节点完成必须是一个原子提交。

## 4. 通用规则

1. 文案中的“商店阶段”只指实际进入显式商店节点并成功开启的 `ShopTurn`。战斗、事件、锻造、恢复和 Boss 重试均不计数。
2. “每经过 N 个商店阶段”从获得遗珍后的下一个商店开始计数，不追溯获得前的 `ShopTurn`。
3. 自动获得的法术或随从进入现有 FIFO 卡牌奖励队列；备战区满时不会丢失。
4. 自动获得的随从必须从有限牌池预留 1 个副本。没有合法副本时不生成卡牌并保留已满进度，下个商店再次尝试。
5. 自动获得的法术只从已启用、`Playable`、商店可出现且不高于当前酒馆等级的普通法术中抽取；排除三连奖励等系统法术。
6. 候选、自动卡牌和战斗随机目标使用独立的确定性随机流，不消费商店商品或遭遇的随机流。
7. 每件遗珍每个逻辑时机最多结算一次；场景重载、重复回调和 Boss 重试不能重复触发。
8. 遗珍效果只作用于玩家侧，除非文案明确说明。
9. 同名遗珍不可重复，v0.1 所有效果均不叠层。

### 5.1 B01 回魂丧钟

- 只增加亡语效果的触发次数，不额外产生死亡、友方死亡或 Token 死亡事件。
- 普通、金色和召唤物的亡语都适用。
- 每次额外触发独立解析随机目标，但使用同一死亡来源快照。
- 额外触发不会再次被本遗珍复制，固定从 1 次变为 2 次。
- 每个效果自身的 `perCombat` 限制仍跨两次触发累计；实施前必须审计所有带隐藏上限的亡语，避免文案承诺与实际表现不一致。

### 5.2 B02 双生战号

- 只重复拥有 `Battlecry` 关键词的随从的 `OnPlay` 效果；打出动作、打出统计、三连处理和其他随从观察到的打出事件仍只有一次。
- `PlayerChoice` 目标复用玩家第一次选择的目标。
- 随机目标在第二次执行时重新抽取。
- 发现、复制等阻塞选择依次排队，前一个完成后再打开下一个。
- 金色战吼同样额外触发一次，不把普通效果降级后再重复。

### 5.3 B03 无价金券

- 随从和法术共享每个商店阶段的一次免费购买。
- 只在一次购买成功后消耗；余额不足、备战区满、非法索引或阻塞选择中的失败请求不消耗。
- 卡牌奖励领取、酒馆升级、刷新和冻结不属于购买。
- `OnBuy` 事件记录的实际费用为 0，统计仍记录一次购买。

### 5.4 B04 星海秘囊

- 在商店预算和延迟资源应用完成后、玩家获得操作权前触发。
- 卡牌进入 FIFO 奖励队列，不直接绕过备战区容量和领取流程。
- 每个 `ShopTurn` 最多生成一次；重载同一商店不再次生成。

### 5.5 B05 五阶征召令

- 获得后进度为 0；未来每个商店开始时增加 1。
- 进度达到 2 时，从所有启用的 5 级非 Token 随从中按有限牌池抽取 1 张并清零。
- 没有可预留副本时保留 2 点进度，下个商店再次尝试。
- 酒馆等级不限制该奖励，这是冠冕级遗珍的主要强度来源。

### 5.6 B06 千盾王冠

- 只选择开战快照中存活、没有永久或下场护盾的随从。
- 按当前生命升序、槽位升序选择最多 2 个目标。
- 护盾持续到本场战斗结束，可以触发已有的获得护盾与失去护盾联动。

### 5.7 B07 荒野母巢

- 每场战斗只响应第一次友方非 Token 死亡。
- 使用现有 `token_young_spirit`，由遗珍将本次召唤的基础属性覆盖为 2/2，并从死亡后产生的空位开始按现有召唤规则放置。
- 召唤失败遵循普通召唤失败规则，不提供额外补偿。
- 幼灵死亡不会再次触发本遗珍。

### 5.8 B08 万象军旗

- 只统计开战快照中友方非 Token 的四个正式种族；无种族单位不增加 X。
- 所有友方开场单位获得本场属性，Token 不会在之后补领。
- X 为 1–4；空阵容时不触发。
- 属性只在本场战斗生效，不写回永久成长。

### 6.1 奇物精确规则

- E01 只在刷新最终计算结果需要支付金币时触发。已有免费刷新次数优先消费，因此免费刷新不会提前消耗漏刻齿轮。
- E02 在基础预算、计划金币和事件延迟金币全部应用后检查 `Health × 2 <= MaxHealth`；满足时增加 2 金币并计入当前商店可用金币。该条件使其成为低生命追赶能力，不再严格支配 E01。
- E03 只响应成功出售的随从；法术移除、三连素材消耗和系统清理不算出售。普通或金色随从均只额外获得 2 金币。它在发生出售时提供比常规单次经济遗珍更高的主动变现收益，避免原 +1 版本被其他经济奇物严格支配。
- E04 只响应玩家胜利的精英或 Boss 节点，普通胜利、失败和平局不恢复。恢复发生在战斗伤害结算后、奖励选择前且不超过最大生命。
- E05 与 B06 同时持有时合并为最多 3 个目标，按生命和槽位统一排序，避免两个效果命中同一个单位。
- E06 与 B04 可以同时存在，各自维护触发记录；同一商店最多因此获得 2 张法术。
- E07 使用当前酒馆等级精确抽取并预留有限牌池副本；没有副本时保留已满进度。

## 7. 结算时序与组合规则

### 7.1 商店开始

一次实际商店开始按以下顺序执行：

1. `ShopTurn` 幂等校验与基础预算。
2. 计划金币、免费刷新和升级折扣等延迟资源。
3. E02 行商零钱袋。
4. B04、B05、E06、E07 的进度与卡牌奖励生成。
5. 普通商店阶段开始事件和随从监听效果。
6. 打开 FIFO 奖励领取和玩家操作。

### 7.2 购买、刷新与出售

- 购买先计算 B03，再验证当前金币和容量；成功后才提交免费次数。
- 刷新先消费显式免费刷新次数，再检查 E01，最后才支付普通刷新费用。
- 出售先完成卡牌移除和基础出售金币，再提交 E03 的额外金币。

### 7.3 战斗开始

1. 从持有遗珍生成只读 `BattleRuleModifiers`。
2. 合并 B06 与 E05 的护盾目标数量并应用。
3. 应用 B08 的混编属性。
4. 应用已有法术预定的战斗开始效果。
5. 按槽位执行随从 `OnBattleStart`。

所有战斗相关遗珍必须写入战斗快照或其确定性哈希输入。UI 只能展示该快照，不能在表现层重新决定目标或数值。

### 7.4 触发倍增

- B01、B02 都采用“基础触发次数 + 额外次数”的整数修饰器，v0.1 最大值为 2。
- 倍增只作用于源效果，不复制外围领域事件，避免观察者和统计同时翻倍。
- 效果队列总上限继续有效；命中安全上限视为实现或内容错误，不能静默截断后继续结算。

## 8. 配置模型

新增 `relics.v0.1.json`：

```text
RelicConfigFile
└── relics[]
    ├── id / name / description
    ├── grade              Crown | Curio
    ├── category
    ├── effectType         白名单枚举
    ├── amount / attack / health / interval / targetCount / cardTier
    ├── tokenId / selector / tierMode
    ├── conditionType / threshold
    ├── enabled / implementationStatus
    └── uiIconId / devNote
```

第一版 `effectType` 白名单：

- `ExtraDeathrattleTriggers`
- `ExtraBattlecryTriggers`
- `FirstPurchaseFree`
- `GrantRandomSpellByShopInterval`
- `GrantRandomMinionByShopInterval`
- `GrantCombatShieldAtBattleStart`
- `SummonOnFirstFriendlyNonTokenDeath`
- `GrantCombatStatsPerDistinctRace`
- `FirstPaidRefreshFree`
- `GoldOnShopStart`
- `GoldOnFirstMinionSold`
- `HealAfterEliteOrBossVictory`

不复用卡牌 `EffectConfig` 直接描述整件遗珍。部分实际战斗效果可以在构造战斗快照时转成现有 `EffectConfig`，但遗珍的获取、进度和跨领域生命周期由独立系统负责。

## 9. 运行时状态

```text
├── sourceType / sourceId
├── acquiredFloor / acquiredShopTurn

PendingRelicChoice
├── choiceId / sourceAttemptId
├── grade / completionMode

```

约束：

- 状态保存配置 ID 和必要进度，不复制名称与文案。
- 候选一旦物化即写入状态，重载不得重抽。
- `choiceId`、来源尝试和最后结算回合共同保证幂等。
- 获取后立即重算商店与战斗规则，但不追溯当前已完成的触发时机。

## 10. 领域职责

### 10.1 `RelicService`

由 `RunSession` 持有，负责：

- 候选物化、等级、重复与事件生命代价校验；
- 遗珍获得和进度状态；
- 商店开始、购买、刷新、出售和战斗结算钩子；
- 构造只读 `ShopRuleModifiers` 与 `BattleRuleModifiers`；
- 自动卡牌奖励的确定性选择和有限牌池预留；
- 遗珍遥测。

`RelicService` 不直接操作 UI，也不重新实现商店购买或战斗效果队列。

### 10.2 `RunSession`

- 仍是地图、Boss、事件、生命和奖励选择的唯一公开写入口。
- 第一、二层 Boss 胜利调用冠冕候选流程，取代旧 Boss 奖励表。
- 遗珍事件使用独立 `PendingRelicChoice`，选择时原子扣血。
- 单局结束释放仍在遗珍自动奖励队列中预留的随从副本。

### 10.3 `ShopSession`

- 只消费 `ShopRuleModifiers` 计算实际购买、刷新和出售结果。
- 不读取 `RunState`，不判断遗珍 ID。
- 所有费用和额外金币进入现有 `ShopEventData`，保证遥测与 UI 一致。

### 10.4 `BattleSimulator`

- 只消费战斗快照中的 `BattleRuleModifiers`。
- B01/B07 的全局触发由模拟器队列处理；B06/E05/B08 尽量在快照构造时物化为明确效果。
- 普通模拟、播放模拟和离线批次必须得到相同结果与确定性哈希。

## 11. UI 与信息呈现

### 11.1 常驻显示

- Run 顶栏显示遗珍入口、持有数量和最近获得的遗珍；展开后使用可滚动或换行列表展示全部持有项。
- 每项显示等级、图标、名称和精确文案；带进度的遗珍显示 `1/2`、`2/3` 等状态。
- UI 不能假设固定三个槽位，列表必须能在持有 0–15 件时正常重建。

### 11.2 Boss 选择

- 使用独立“Boss 战利品”标题和三张冠冕级候选卡。
- 不显示跳过按钮。
- 必须选中一件后才能进入下一层。

### 11.3 事件选择

- 在候选卡上明确显示当前生命、代价和选择后的剩余生命。
- 允许在未选择具体遗珍前离开，不扣生命。
- 生命不足时所有购买按钮置灰并显示“至少保留 1 点生命”。

### 11.4 反馈

- 商店日志记录免费购买、免费刷新、额外金币和自动卡牌来源。
- 战斗日志记录遗珍开场效果和 B07 的召唤来源。
- B01/B02 在播放事件中标记“遗珍额外触发”，但不伪造第二次打出或死亡事件。

## 12. 遥测与平衡

### 12.1 评价口径

阶段 8A 实施时的显式地图每层固定包含 5 个商店和 5 场战斗，以下保留为 5.4.1 历史评价基线。阶段 8B 已扩展为 6×6 节奏，当前收益窗口与复核结论以 `phase-8b-full-map-technical-design-v0.1.md` 为准。

| 获得时机 | 剩余商店 | 剩余战斗 |
| --- | ---: | ---: |
| 第一层 Boss 后 | 10 | 10 |
| 第二层 Boss 后 | 5 | 5 |
| 第一层中段事件 | 13 | 13 |
| 第二层中段事件 | 8 | 8 |
| 第三层中段事件 | 3 | 3 |

经济换算使用当前费用：随从 3、法术 1、刷新 1、出售随从基础获得 1。评级表示当前规则下的相对上限：

- `S+`：高概率成为跨构筑优先选择，必须进入首轮重点观察；
- `S`：符合冠冕级强度，能显著改变构筑；
- `A`：符合本等级预期，有稳定收益或清晰构筑条件；
- `B`：收益依赖特定场面，可能需要增强；
- `C`：被同等级其他遗珍明显支配，需要在实施前调整。

### 12.2 冠冕级逐项复核

| 编号 | 初评 | 当前三层下的主要收益 | 结论 |
| --- | --- | --- | --- |
| B01 回魂丧钟 | `S`（亡语）/ `B`（其他） | 最多影响剩余 10 或 5 场战斗；可重复召唤、繁茂、战后卡牌和永久成长 | 上限高但构筑依赖明确，保留 |
| B02 双生战号 | `S+` | 当前有 10 个可用战吼随从；可重复金币、永久属性与发现，金色双发现会进一步放大 | 冠冕级上端，首轮重点观察 |
| B03 无价金券 | `S` | 若每个商店先购买随从，第一/二层获得时最多节省 30/15 金币 | 无条件下限最高，首轮重点观察 |
| B04 星海秘囊 | `S+` | 最多生成 10/5 张法术；当前普通法术池含高阶发现、全体成长和 5 级终结法术 | 高方差且能放大施法构筑，首轮重点观察 |
| B05 五阶征召令 | `S` | 最多生成 5/2 张 5 级随从；当前有 11 种 5 级随从，每种初始 4 份 | 强但受两商店间隔、随机结果和有限牌池约束，保留 |
| B06 千盾王冠 | `A` / 护盾构筑 `S` | 理论最多提供 20/10 次开场护盾，能接入获得/失去护盾联动 | 构筑倾向清晰，保留 |
| B07 荒野母巢 | 原 `B`，调整后 `A` | 每战首次非 Token 死亡提供最多两个 2/2，受空位限制但可触发召唤与 Token 死亡联动 | 原 1/1 明显偏弱，已增强为 2/2 |
| B08 万象军旗 | `A` / 混编 `S` | 五单位两种族通常为全队合计 +10/+10，四种族上限为 +20/+20，均只持续本场 | 上限高但不产生永久经济，保留 |

冠冕级存在三个高强度观察项，但暂不预先削弱用户可感知的核心玩法：

- B02 若在多数非战吼构筑中仍成为首选，优先调整为“每个商店阶段第一个战吼额外触发一次”。
- B03 若跨构筑稳定领先，优先调整为“第一次购买费用减少 2，最低为 0”。
- B04 若高阶法术导致成长失控，优先将随机法术最高等级限制为 3，而不是降低触发频率。

这些是预先确定的降档轴，每轮只能使用其中一个，不同时修改触发频率、候选池和数值。

### 12.3 奇物级逐项复核

| 编号 | 初评 | 当前三层下的主要收益 | 结论 |
| --- | --- | --- | --- |
| E01 漏刻齿轮 | `A` | 若每个商店都至少进行一次付费刷新，最多节省 13/8/3 金币 | 需要实际刷新，收益稳定但不自动发生，保留 |
| E02 行商零钱袋 | `A` | 低于半血期间每商店 +2；理论上限 26/16/6，但需要持续承担高失败风险 | 原无条件 +1 会支配 E01，已改为低生命追赶能力 |
| E03 回收吊钩 | 原 `C`，调整后 `A` | 首次成功出售额外 +2；平均每两个商店出售一次时约等价每商店 +1 | 原额外 +1 被旧 E02 严格支配，已增强为 +2 |
| E04 战地药箱 | `B` / 高风险路线 `A` | 只计剩余 Boss 时至少可恢复 6/4/2，额外精英胜利继续增加 | 回报取决于路线和缺失生命，符合奇物定位 |
| E05 裂纹护符 | `A` / 护盾构筑 `S` | 最多影响 13/8/3 场战斗，每场保护一个单位并提供护盾联动 | 构筑向收益较高但只命中单体，保留 |
| E06 灵感墨瓶 | `A` | 最多生成 6/4/1 张法术，受商店间隔和酒馆等级约束 | 与 B04 有清楚的频率差，允许同时持有 |
| E07 征募铜章 | `A` | 最多生成 4/2/1 张当前等级随从，名义购买价值约 12/6/3 金币 | 受三商店间隔、随机结果和有限牌池约束，保留 |

### 12.4 无持有上限后的组合风险

移除三件上限后，没有单件遗珍因为持有数量变化而立即失效，但下列组合需要单独统计：

- B03 + E01 + E02 + E03：购买、刷新、低生命补助和出售收益叠加，可能形成经济滚雪球。
- B04 + E06：同一商店最多生成两张免费法术，会同时放大星契施法成长。
- B05 + E07：随从奖励加速消耗有限牌池，也提高三连概率。
- B06 + E05：开场最多为三个不同单位授盾，可能大幅提高铸魂永久成长。
- B01 + B07：额外亡语和全局召唤同时提高荒灵的召唤、死亡与永久成长密度。
- B02 + B03：免费购买降低循环战吼随从的成本，是目前最高风险的跨系统组合。

v0.1 不为这些组合设置人工互斥组。优先通过事件密度、生命成本和候选随机性控制获得数量，并用组合遥测判断是否需要后续软上限或互斥标签。

### 12.5 当前强度结论

- 明显偏弱且已调整：B07 从两个 1/1 提高为两个 2/2；E03 首次出售额外金币从 1 提高为 2。
- 存在同级支配且已调整：E02 从无条件每商店 +1 改为半血以下每商店 +2，使其与 E01 分别承担追赶和刷新构筑定位。
- 高强度但暂不削弱：B02、B03、B04；三者分别代表战吼、通用经济和法术构筑的冠冕级上限。
- 其余 9 件没有发现明显的严格支配或无条件强度断层，可以进入首轮无 Unity 回归和小样本模拟。

### 12.6 遥测

新增遥测：

- 候选、选择、来源、楼层和获得时的 `ShopTurn`；
- 事件遗珍支付生命；
- 每件遗珍的实际触发次数、生成卡牌数、节省或获得金币、恢复生命；
- B01/B02 的额外效果执行次数；
- B08 每场 X 值；
- 最终完整遗珍列表、获取顺序、同时持有数量峰值与单局结果。

### 12.7 初始平衡目标

初始平衡目标：

- 奇物级以每个剩余商店约 1 金币等价或单目标战斗收益为基准；
- 冠冕级允许达到每商店约 3 金币等价，或显著改变一个构筑机制；
- 第一层事件遗珍因剩余消费节点多而支付更高生命；
- 候选优先不同类别，避免三张纯经济或三张纯战斗候选造成假选择。

遗珍会改变经济、战斗、永久成长和卡牌获得速度。当前已建立 5.4.1 内容版本与完整配置哈希，Unity 全量回归和交互式人工单局已经通过，并纳入新的运行时候选。R17 的 S0 只能保留为遗珍系统前的历史证据，不得继续补跑其 S1/S2。

## 13. 自动化测试

### 13.1 配置与候选

- 恰好 15 件启用遗珍：8 件 `Crown`、7 件 `Curio`。
- ID 唯一，等级、类别、效果类型和参数组合合法。
- Boss 候选只含冠冕级，事件候选只含奇物级。
- 已持有项不重复出现，三选一优先不同类别。
- 持有数量增加不会阻止继续获得其他遗珍；只有同名重复或候选池耗尽会阻止生成。

### 13.2 Boss 与事件

- 第一、二层 Boss 胜利必生成且只生成一次不可跳过的三选一。
- Boss 失败、重试、重复提交和场景重载不提前或重复生成遗珍。
- 第三层 Boss 不生成无消费价值的局内遗珍。
- 事件候选在选择前可离开且不扣血。
- 事件选择原子扣血并获得遗珍，生命至少保留 1。

### 13.3 商店

- B03 只免除每商店第一次成功购买，失败请求不消耗。
- E01 在普通免费刷新之后生效且每商店一次。
- E02、E03 金币与事件统计一致。
- B04/B05/E06/E07 的进度、奖励队列、牌池预留和重载幂等正确。
- 无合法卡牌时保留进度，不生成空奖励或凭空副本。

### 13.4 战斗

- B01 对普通、金色、Token 亡语生效，但死亡事件和统计仍为一次。
- B02 对指向、随机、发现和金色战吼分别符合精确规则。
- B06 与 E05 合并目标数量且不重复授盾。
- B07 每场只响应首个非 Token 死亡，幼灵不会递归触发。
- B08 的 X 为 1–4，属性仅本场有效。
- 普通模拟、播放模拟、同种子复跑和确定性哈希一致。

### 13.5 组合回归

- B01 + B07 不把遗珍召唤识别为亡语重复。
- B03 + E01 分别只修改购买与刷新。
- B04 + E06 同商店可以稳定生成两条独立法术奖励。
- B05 + E07 的有限牌池预留与单局结束返还守恒。
- B06 + E05 最多命中三个不同的合法目标。

1. 增加遗珍配置、15 件内容和加载/校验。
2. 增加 `OwnedRelicState`、`PendingRelicChoice` 与独立随机流。
3. 接通 Boss 强制三选一和事件原子生命交易。

10. 制作动态遗珍列表、候选卡和事件生命代价反馈，完成静态验收后再执行交互式人工单局。

## 15. 完成定义

- 15 件遗珍全部配置化、可校验并具有唯一精确文案。
- 单局遗珍无持有数量硬上限，同名遗珍不可重复。
- 第一、二层 Boss 必然获得一件冠冕级遗珍，失败重试不能刷候选。
- 事件可以在不致死的前提下支付生命获得一件奇物级遗珍。
- 所有商店与战斗效果按确定性、幂等和有限牌池规则运行。
- 动态遗珍列表、Boss/事件选择和进度反馈可从状态重建。
- 全量自动化通过并建立新的内容与运行时候选身份。

## 16. 当前结论

v0.1 采用 8 件冠冕级加 7 件奇物级，共 15 件。冠冕级承担“改变规则”的强效果，奇物级承担“提高效率”的辅助效果。系统不设置持有数量硬上限；当前三层流程仍会通过两个可消费 Boss 和有限事件数量形成自然获取上限，完整地图则必须把遗珍事件密度和生命代价纳入路线收益预算。

该系统应先于完整地图的最终定稿实施或至少冻结规则，因为完整地图需要据此重新评估事件密度、精英/Boss 路线收益和生命资源价值。
