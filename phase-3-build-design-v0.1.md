# 阶段 3 构筑系统设计 v0.1

版本：0.1
状态：已确认开发基线；阶段 3A/3B 领域逻辑和最小 Unity 交互已实现，自动化测试通过，待 Unity 手动交互验收
关联文档：`game-design.md`、`minion-design-v0.1.md`、`spell-design-v0.1.md`、`shop-economy-design-v0.1.md`

## 1. 阶段目标

- 让商店中购买的随从和法术能够产生实际构筑价值。
- 用统一效果执行入口支持战吼、法术、资源效果和发现。
- 用自动三连、金色随从和打出金色随从时获得的发现法术制造明确的构筑兴奋点。
- 继续保持纯领域逻辑，不依赖 Unity 场景和 UI。

阶段 3 第一批只实现必要动作，不提前实现完整随从效果 DSL、光环和战斗中手动施法。

## 2. 商店、阵容与战斗衔接

领域层提供显式的商店阶段开始和结束入口，地图或流程层决定何时推进经济回合。

推荐流程：

1. 开始商店阶段，重置本回合预算并生成或补充商店。
2. 玩家购买、出售、刷新、升级、冻结、使用法术和调整阵容。
3. 结束商店阶段，生成当前战斗区的战斗快照。
4. 战斗只修改快照中的临时状态，不因死亡移除玩家持有的随从。
5. 战斗结束后，只把明确标记为永久的变化同步回持有阵容。

备战区和战斗区中的随从、法术实例属于单局持有状态；战斗运行时对象不直接替代这些持有实例。

- 阶段 3 将商店阶段同时视为阵容准备阶段，所有法术使用和站位调整必须在结束商店前完成。
- 暂不建立独立的战前准备阶段；结束商店后阵容锁定并直接生成战斗快照。
- 只有战斗区中的非 Token 随从可以在商店阶段出售；备战区中的未打出随从不能出售。

## 3. `OnPlay` 战吼规则

### 3.1 触发时机

- 战吼统一绑定 `OnPlay`。
- 玩家主动将随从从备战区放入战斗区时触发 `OnPlay`。
- 购买随从进入备战区不触发 `OnPlay`。
- 战斗区内部换位不触发 `OnPlay`。
- 战斗区随从可以在战斗区内部换位，但不能由玩家手动收回备战区。
- 只有三连合成、明确写明“返回备战区”的卡牌效果或其他特殊规则，才能使战斗区随从回到备战区。
- 通过特殊规则回到备战区的随从再次打出时，会再次触发 `OnPlay`。
- 三连生成的金色随从进入备战区时不触发 `OnPlay`；之后实际打出时才触发金色战吼。
- 三连生成的金色随从首次成功打出时，同时把一次性的发现法术奖励放入备战区。
- 该奖励不是可重复触发的战吼；金色随从收回后再次打出只会再次触发其金色战吼，不会重复获得发现法术。

### 3.2 上场与效果结算

- 上场和战吼作为一次原子操作处理。
- 需要玩家选择目标时，先验证目标，再改变位置和执行效果。
- 备战区随从只能打入空战斗位；目标战斗位已有随从时上场失败，不允许借此把原随从交换回备战区。
- 战吼没有合法目标时，随从仍正常上场，仅跳过没有合法目标的效果。
- 随机目标效果在执行前从所有合法目标中等概率选择；没有合法目标时跳过该效果。
- 有合法候选但玩家提交了非法目标时不开始操作，随从仍留在原备战区位置。
- 打出仍有发现奖励待领取的金色随从时，位置变化后必须有足够空格容纳发现法术及该战吼产生的其他卡牌；空间不足时整次上场失败。
- 多个战吼效果按配置顺序结算；没有合法目标的单个效果可以跳过，其他合法效果继续结算。
- 普通随从执行 `effects` 中的 `OnPlay`；金色随从执行 `goldenEffects` 中的 `OnPlay`，不重复执行普通版本。

### 3.3 目标范围

- 阶段 3 的战吼和法术都只能选择操作完成后的最终战斗区，不能选择备战区随从。
- 战吼文本中的“友方随从”默认不包含战吼来源自身。
- 需要作用于战吼来源时，配置必须显式使用 `scope: Self`。
- 法术没有随从来源；只要目标位于最终战斗区并满足其他筛选条件，即可成为法术目标。

### 3.4 第一批支持动作

- `ModifyStats`：永久修改自身或合法友方随从的攻击和生命。
- `GainGold`：立即获得金币，允许当前金币超过 10。
- `FreeRefresh`：增加免费刷新次数。
- 使用 `TargetConfig` 支持自身、指定友方和随机友方目标。

第一批验收随从固定为：

- 观星学徒：`OnPlay` 获得 1 次免费刷新。
- 黑市小贩：`OnPlay` 获得 1 金币。

## 4. 统一效果执行规则

战吼与法术共用效果验证和执行入口：

1. 收集效果和候选目标。
2. 验证阶段、目标、条件、收益和空间。
3. 确定随机选择结果。
4. 一次性提交位置、属性、金币、刷新次数和卡牌消耗变化。
5. 完整结算本次操作产生的 `OnBuy`、`OnPlay`、`OnSpellUsed` 等玩法响应。
6. 所有响应结束后检测并逐个结算三连；每次合成后重新检测。
7. 状态稳定后发出供 UI、日志和调试使用的通知事件。

失败操作不得部分生效。第一版不要求通用事务回滚，而是要求所有可能失败的条件在提交前完成验证。

- 玩法响应由效果执行器管理，不允许 UI 通知监听器直接改变领域状态。
- 第三个普通随从在本次操作及响应中获得的永久强化，会在随后三连时正常继承。

## 5. 法术使用规则

- 第一版只支持在商店阶段主动使用法术，商店阶段同时承担阵容准备功能。
- 暂不建立或支持独立的战前准备阶段。
- 战斗开始后不能手动使用法术。
- 法术必须先通过阶段、目标、条件和收益验证，成功后才从备战区移除。
- 目标不存在、条件不满足、没有实际收益或奖励无处放置时，法术保留且状态不变。
- 使用成功后释放原备战区格子并触发 `OnSpellUsed`。
- 多效果法术按配置顺序结算，但提交前必须保证整张法术均可合法结算。

第一批实现 4 类基础法术：

- 小型锻体：指定最终战斗区随从永久获得 +1/+1。
- 精准训练：指定最终战斗区随从永久获得 +2 攻击。
- 厚皮药剂：指定最终战斗区随从永久获得 +3 生命。
- 免费刷新：本商店阶段获得 2 次免费刷新。

## 6. 三连检测

### 6.1 统计范围

- 战斗区和备战区中的同配置普通随从共同计数。
- Token、金色随从和商店中尚未购买的随从不计入三连。
- 三个相同普通随从满足条件时自动合成，不提供手动确认步骤。
- 超过三个时每次只消耗三个，剩余副本继续保留并重新检测。

### 6.2 消耗顺序

需要从多于三个同名普通随从中选择材料时，固定按以下顺序：

1. 备战区从左到右。
2. 战斗区从左到右。

固定顺序仅用于保证结果可预测，不影响三连所需的副本总数。

- 同时存在多个可合成配置时，按“备战区从左到右、再战斗区从左到右”首次出现的配置顺序结算。
- 每次只合成一个金色随从，随后从最新持有状态重新扫描，直到没有可立即结算的三连。

### 6.3 备战区空间

- 三连结算时只生成金色随从，因此材料移除后至少有 1 个备战区空格即可合成。
- 空间足够时自动合成，金色随从进入材料移除后的最左空格。
- 空间不足时不消耗任何材料，记录为等待合成状态。
- 购买、出售、移动、使用法术等操作改变备战区空间后，重新检测等待中的三连。
- 不允许为了完成三连丢弃法术或随从。

## 7. 金色随从

- 三个普通随从合成为一个金色随从实例，并标记 `IsGolden=true`。
- 金色基础攻击和生命使用配置中的 `goldenAttack`、`goldenHealth`，不在运行时套用统一翻倍公式。
- 金色效果完整使用 `goldenEffects`，它是普通效果的替代版本。
- 三个材料的永久攻击、永久生命加成相加后继承到金色实例，永久关键词取三个材料的并集。
- 金色实例同时保留配置自身的基础关键词，并叠加继承的永久关键词。
- 护盾当前或已消耗状态、临时战斗属性、仅限下一场战斗的状态和触发次数不参与继承。
- 合成过程不触发 `OnPlay`，金色随从之后从备战区上场时才触发金色战吼。
- 三连生成的金色随从带有一次性的 `TripleDiscoveryPending` 状态；首次成功打出并生成发现法术后清除该状态。
- 通过其他奖励直接获得的金色随从默认没有 `TripleDiscoveryPending`，除非具体奖励明确说明。

牌池规则：

- 合成不会把三个材料副本返还有限牌池；金色随从继续代表并占用三个普通实体副本。
- 金色随从必须先打入战斗区才能出售；首次打出时会先正常发放待领取的发现法术。
- 出售战斗区中的金色随从获得 1 金币，并向有限牌池返还三个对应普通副本。
- 金色随从不能与普通随从继续进行三连。

## 8. 金色随从发现法术

- 三连本身不生成发现法术。
- 三连生成的金色随从首次成功从备战区打出时，生成一张专用发现法术并放入最左侧空备战位。
- 发现法术成功进入备战区后清除该金色随从的待领取状态；以后收回再打出不会重复生成。
- 若上场结算后没有空位容纳发现法术，则上场、战吼和奖励均不结算，金色随从仍留在备战区并保留待领取状态。
- 该奖励法术不是商店商品，不占用商店法术池抽取结果，也不产生购买事件。
- 奖励法术使用普通法术配置表示，但必须设置 `shopEligible: false`，不能被商店随机抽取。
- 专用配置 ID 固定为 `triple_discovery_reward`，动作使用 `DiscoverMinion`，等级模式使用 `ExactCurrentTavernTier`。
- 15 张普通法术仍组成商店法术池；该奖励是额外的第 16 张系统法术配置。
- 奖励法术与其他法术一样占用备战区，不能出售或丢弃。
- 使用后从 3 个不同的随从候选中选择 1 个，候选等级必须等于使用法术时的当前酒馆等级；获得法术后升级酒馆再使用，会按升级后的等级生成候选。
- 候选只包含 `enabled=true`、`isToken=false` 且有限牌池仍有副本的随从。
- 每个合法的剩余实体副本等概率参与抽取；某个配置成为候选后，本次发现不再重复展示该配置。
- 合法候选不足 3 个时展示全部候选，不用重复候选补齐。

发现结算：

1. 使用法术时从有限牌池暂时保留候选各一个副本。
2. 进入阻塞式发现状态；完成选择或取消前，不能执行购买、出售、移动、使用其他法术、刷新、升级、冻结或结束商店。
3. 玩家选择后，法术被消耗，选中的随从进入该法术释放出的备战位。
4. 未选择候选立即返还有限牌池，选中的副本继续由玩家持有并占用牌池。
5. 取消选择或结算失败时，全部候选返还牌池，法术保留。
6. 没有任何合法候选时不能使用该法术，也不会进入阻塞状态。

## 9. 开发顺序

阶段 3 拆成两个可独立验收的增量：

- 阶段 3A：持有实例、商店/阵容准备阶段、目标规则、原子 `OnPlay`、统一效果执行器、2 个战吼和 4 张基础法术。
- 阶段 3B：自动三连、金色状态继承、一次性发现奖励、阻塞式发现和有限牌池候选保留。

当前实现状态：阶段 3A 与 3B 的纯 C# 领域逻辑均已完成；Unity 场景、动画、拖拽和发现选择界面仍待统一环境接入。

阶段 3B 对外通知顺序：

- 普通操作先完成所有状态提交和递归三连，再依次发出本次操作事件及 `OnTripleFormed`。
- 金色首次打出依次发出 `OnPlay`、`OnTripleRewardGranted`，再发出该操作产生的三连事件。
- 开始发现时发出 `OnDiscoverStarted`；选择后依次发出 `OnSpellUsed`、`OnDiscoverResolved` 和后续三连事件；取消时发出 `OnDiscoverCancelled`。
- 所有发现开始通知发出时阻塞状态已经生效，选择或取消通知发出时阻塞状态已经解除。

具体顺序：

1. 增加持有随从的永久属性和金色状态。
2. 建立商店阶段到战斗快照的协调入口。
3. 实现原子上场与 `OnPlay`。
4. 实现统一效果验证器和第一批基础动作。
5. 接入 2 个战吼随从和 4 张基础法术。
6. 实现三连检测、金色继承和有限牌池占用。
7. 实现金色随从首次打出奖励和纯逻辑三选一。
8. 最后接入 Unity 交互和表现。

## 10. 验收标准

- 购买进入备战区不触发战吼，从备战区打入空战斗位时触发。
- 战斗区随从不能手动回收；通过明确的特殊规则返回后再次上场可以再次触发战吼。
- 没有合法目标的战吼不阻止随从上场；没有实际收益的法术不能使用。
- 战吼和法术只能选择最终战斗区；“友方随从”不包含战吼来源，除非显式使用 `Self`。
- 备战区随从不能通过打入已有随从的战斗位把该随从交换回备战区。
- 金色随从执行金色战吼，不同时执行普通战吼。
- 战斗区和备战区的三个同名普通随从可以自动合成。
- 空间不足时三连延迟，空间足够后自动完成且不丢失卡牌。
- 三连时只生成金色随从；首次成功打出该金色随从时，发现法术进入备战区。
- 通过特殊规则返回备战区并再次打出同一金色随从时，不会重复获得发现法术。
- 发现法术没有容纳空间时，金色随从上场失败且所有状态不变。
- 金色随从正确继承三个材料的永久属性，并继续占用三个牌池副本。
- 备战区随从不能出售；战斗区普通随从出售返还 1 个副本，金色随从出售返还 3 个副本。
- 发现法术只产生使用时当前酒馆等级的合法随从候选。
- 发现选择、取消和候选不足时的牌池数量均正确。
- 发现选择期间所有其他商店操作均被阻止。
- 第一批战吼和法术能通过 EditMode 测试及纯 C# 测试桩验证。
